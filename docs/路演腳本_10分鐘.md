## 1. 痛點開場（1.5 分）

> 「各位評審好。請大家諗一個畫面：星期六下午嘅大三巴，遊客逼到水洩不通；
> 但係行落山五分鐘嘅福隆新街、十月初五街——嗰啲真正有澳門味嘅老街——卻冷冷清清，
> 老字號一間間執笠。」

- 澳門旅遊高度集中：少數熱點逼爆，舊區小店凋零。
- 這同時傷害**遊客體驗**、**社區存續**、**城市承載**。
- 一句問題：**有冇辦法用 AI，把人流「重新分配」？**

## 2. 我們的方案（1 分）

> 「我哋做咗『街知巷聞』——一個會**做任務**嘅澳門深度遊 AI 智能體，人設叫『阿濠』。
> 佢唔係問答機械人，而係識得**自己規劃、用工具、改線**，仲會主動將你由熱點，
> **導流入舊區老街同本地小店**。」

## 3. 現場 Demo（4 分 · 核心）

**操作：** 點一個範例 chip —— 「我想去鄭家大屋同附近嘅歷史老街，星期三去」。

一邊講解左邊**實時智能體軌跡**：

1. 「睇住左邊——阿濠**先做計劃**，然後**自己調用工具**：查天氣⋯⋯」
2. 「佢去 `check_opening` 核實鄭家大屋——**發現星期三休息**！注意：佢**冇當機**，
   而係**自動改線**，喺同區揾返一個有開嘅替代點——呢個就係**失敗恢復**。」（指住 🔁）
3. 「跟住佢 `predict_crowd` 預測大三巴正午**極擁擠**，於是 `find_local_gem`
   **加插附近嘅草堆街**分流——呢個就係我哋嘅**核心創新：舊區導流**。」（指住 ↪️）
4. 「最後 `compute_route` 排路線、`estimate_budget` 計預算。」

切去右邊**結果**：

5. 「成份行程**逐項可驗證**：地圖路線、時間軸、人均預算、步行距離。
   呢個『**任務完成核對**』面板——預算、步行、開放、避開人潮、帶旺舊區——
   每一項都打咗剔。」
6. 點開一張卡的「**聽阿濠講古**」，展示在地粵語敘事；切換語言示範多語言。

> 💡 提示：Demo 前先試跑一次，確保網絡正常；若擔心現場網絡，離線引擎同樣完整演示。

## 4. 技術亮點（1.5 分）

- **真·ReAct 智能體**：規劃 → 7 種工具調用 → 多步執行 → 失敗自動恢復。
- **對標 QwenPaw 五層架構**：FastAPI 運行時 + ReAct 核心 + Toolkit + Qwen（百煉 OpenAI 相容接口）。
- **結果可驗證**：工具負責算事實，模型負責做決策，確定性組裝器產出行程——唔靠模型「吹水」。
- **穩健**：在線 Qwen / 離線雙腦，任何環境都跑得起。

## 5. 商業價值（1 分）

- **遊客**慳時間、避人潮、玩得地道；**舊區小店**得到真實新客流；**城市**旅遊更平衡。
- 變現：商戶精選訂閱 / 引流分成、酒店與旅行社 API、文旅數據儀表板、多語言客群。

## 🙋 預期問答（Q&A 準備）

**Q：人流 / 天氣係咪真實數據？**
A：目前以季節與時段模型估算，介面已明確標示；架構已預留接口，可即時接入真實 API。關鍵事實（開放時間、坐標）則為真實、可驗證。

**Q：點解唔直接用大模型生成行程就算？**
A：因為要「任務完成度」同「可驗證」。我哋讓工具計算事實、模型只做決策，避免模型杜撰開放時間或亂排路線；輸出由確定性組裝器產生，穩定可靠。

**Q：同 QwenPaw 關係？**
A：作品對標 QwenPaw 的五層架構與 ReAct + Toolkit 理念，並透過百煉 OpenAI 相容接口接入 Qwen；後續可直接落地為 QwenPaw 上的 Skill + 定時任務。

**Q：點樣賺錢 / 商戶點上線？**
A：B 端商戶精選訂閱與引流分成為主；商戶端可上傳優惠、查看引流統計，平台按帶客量計費。

**Q：擴展性？**
A：景點知識庫為結構化 JSON，新增景點即可；多語言、多日行程、真實 API、商戶端均已在路線圖內。

**Q：點解叫「街知巷聞」？**
A：粵語成語，意思係「每條街每條巷都知道」——正好對應我哋帶遊客行勻每一條老街嘅使命。
